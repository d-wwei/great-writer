## 一句话总结

**一套 AI 写作协议，专治技术产品文章的"AI 味"。** 数据驱动、场景化叙事、节奏感强——写出来的文章像人写的，不像 AI 吐的。

## 它解决什么问题

让 AI 写技术产品文章，你大概率会收到这种东西：

> "随着人工智能技术的快速发展，XX 产品凭借其强大的功能和出色的性能，为用户带来了前所未有的体验..."

**没人想读这种文章。** 没有数据、没有痛点、没有画面感，纯粹的 AI 套话堆砌。

Great Writer 是一套写作协议（Skill），加载后 AI 会遵循一套经过验证的写作框架：

| 问题 | Great Writer 的解法 |
|------|-------------------|
| 空洞无物 | 每篇文章必须有 1-2 个"杀手数据"作为锚点 |
| AI 味重 | 内置套话黑名单，禁止"随着"、"众所周知"等 |
| 功能罗列 | 强制按使用场景分组，配场景示例 |
| 读不下去 | 痛点开场、短段落、反问句制造节奏 |
| 概念晦涩 | 类比优先于解释 |

## 核心写作框架

### 4 条铁律

1. **数据是锚点** — "省下 96% Token" vs "大幅提升效率"，前者能被转述，后者不能
2. **痛点开场** — 第一段进入读者的痛，不暖场、不铺垫
3. **类比大于解释** — "MCP 是背着 1350 件套工具箱出门" 胜过三段技术描述
4. **场景化展示** — "AI 自动搜文档、写周报" 而不是 "支持文档搜索、文档创建"

### 9 模块文章结构

```
标题（必须含数字）
  → TL;DR（一句话说清价值）
    → 痛点切入（数据 + 后果）
      → 数据对比表（三方对比，🔴🟡🟢）
        → 原理类比（一个记得住的比喻）
          → 功能全景（按场景分组 + 示例）
            → 兼容性
              → 快速上手（5 步以内）
                → 收尾（回扣数据 + 行动号召）
```

### 自检清单

写完逐项过：

- [ ] 标题有数字吗？
- [ ] 前 3 句能独立成一条朋友圈？
- [ ] 核心数据出现了至少 3 次？
- [ ] 有一个让人记住的类比？
- [ ] 功能是按场景展示还是列清单？
- [ ] 有没有 AI 味套话？
- [ ] 你自己会转发这篇吗？

## 适用场景

| 平台 | 字数建议 |
|------|---------|
| 微信公众号 | 1200-1800 字 |
| 技术博客 | 1500-2500 字 |
| 产品 Landing Page | 500-800 字 |
| Twitter/X Thread | 每条 <100 字，5-8 条 |

## 安装

### Claude Code

```bash
# macOS / Linux
ln -sf /path/to/great-writer/skill ~/.claude/skills/great-writer

# 或者直接克隆
git clone https://github.com/d-wwei/great-writer.git
ln -sf "$(pwd)/great-writer/skill" ~/.claude/skills/great-writer
```

### 其他 Agent

将 `SKILL.md` 的内容作为 system prompt 或上下文注入即可。核心是那套写作协议，不依赖特定框架。

## 使用

安装后，对话中提到以下关键词会自动触发：

> "写文章"、"公众号"、"技术博客"、"产品介绍"、"write article"、"blog post"

示例对话：

```
你：帮我写一篇公众号文章介绍我们的新产品，核心卖点是速度快 10 倍、价格低 80%
AI：[自动加载 great-writer skill，按 9 模块结构输出文章]
```
