# Great Writer

给 AI Agent 装一套写作系统。不是提示词模板——是一条 6 阶段管线，逼 Agent 先想透再动笔，写完再逐层刮掉 AI 味。

9 种写作模式。15 条写作原则。4 轮打磨。中英双语。

---

## 为什么做这个

你让 AI 写东西，收到的第一句：

> *"随着人工智能技术的快速发展，XX 产品凭借其强大的功能..."*

模型没坏。缺的是写作**系统**。

市面上的 AI 写作工具大多是在给你更好的模板——更精致的菜谱卡片。产出确实好了一点，但读起来还是像 AI。句子都一样长。段落完美对称。每个列表刚好三项。读者一眼就看穿了。

问题不在格式。在于 AI 压根没**想过**它在写什么。从你的提示词直接跳到成品，中间什么都没有——没调研、没质疑、没验证核心观点站不站得住。

Great Writer 补上的是专业写手真正花时间的部分：搞清楚到底什么值得写（然后攻击这个观点直到它扛住），再系统性地刮掉每一个"机器写的"痕迹。

打个比方：大多数 AI 写作工具是菜谱集——照着做，出标准菜。Great Writer 是厨师训练——教你为什么这样切、这样火候、这个食材配这个时间。菜谱集出不了好厨师，厨师训练能。

---

## 这是什么

一个 Skill 文件，装到任何 AI Agent（Claude Code、Codex、Cursor、Windsurf、Gemini CLI）就能用。

装完你的 Agent 多了三样东西：

- **9 种写作模式** — 技术文章、营销文案、研报、小红书笔记、GitHub README、技术文档、改写润色、编辑审稿、创意写作
- **6 阶段管线** — 调研 → 找核 → 搭结构 → 写稿 → 审校 → 打磨 → 定稿
- **4 轮打磨系统** — 口语检验、密度压缩、AI 痕迹清除、反风格检查

一个 Skill 目录（~20 个文件）。或者一个合并版文件，不支持多文件的 Agent 也能用。

---

## 核心特性

**找核 → 攻核。** 写之前，Agent 用四把铲子挖出这篇文章真正要说的东西——反转判断、追问前提、追问情绪、翻转定义。然后攻击这个核："如果这是真的，那为什么……？"核碎了就老实告诉你，不会硬撑着写一篇站不住的文章。别的写作工具不做这一步。

**4 轮打磨，合并两套方法论。** 第 1 轮：口语检验（"你会这样跟朋友说吗？不会就改"）。第 2 轮：五维密度压缩（砍、短、造、选词、节奏）。第 3 轮：AI 痕迹清除（中英双语黑名单 + 8 种结构模式检测 + 翻译腔免疫）。第 4 轮：反风格检查（在解释？→ 换成场景。在罗列？→ 只留最狠的一个）。

**按模式调声音。** 每种写作模式有自己的温度、姿态和生效原则。小红书笔记 35°C，打开内心声音，"姐妹们听我说"的亲密感。技术文档 18°C，零修辞疑问，每个字只为清晰服务。同一个 Skill 写出真正不同的声音——不是一个模板微调。

**强制调研，3 层痛点深挖。** 不出 Research Summary 不许动笔。痛点必须挖到第三层——不是"问题存在"（L1），不是"有什么后果"（L2），而是"人们为了绕开这个问题做了什么荒唐事"（L3）。这就是为什么产出是具体的而不是泛泛的。

**15 条写作原则，可机械检查。** 不是"写好一点"这种模糊建议。而是：不许 3 句连续等长。不许段尾膨胀意义。不许同一种句式出现两次。每条规则有测试标准——过了就过了，没过就改。

## 有没有 Great Writer 的区别

| | 没有 Great Writer | 有 Great Writer |
|---|---|---|
| **写之前** | Agent 直接开始写 | Agent 调研受众、找核、攻击核心 |
| **核心观点** | 提示词暗示什么就写什么 | 四把铲子挖出来，攻击验证过 |
| **结构** | AI 默认：开头 → 正文 → 结尾 | 刻意设计：重点 40%，背景 5% |
| **初稿质量** | 有多少写作原则在生效？零 | 15 条原则 + 模式声音加载 |
| **AI 味** | 祈祷读者看不出来 | 4 轮系统：黑名单 + 结构检测 + 口语检验 |
| **中文质量** | 翻译腔遍地 | 翻译腔免疫：症状清单 + 回译测试 |
| **下次写** | 从零开始 | 写作记忆：品牌词、数据、风格跨会话保留 |

## 工作原理

### 6 阶段管线

不管写什么类型，都走同一条管线：

```
Phase 1          Phase 2          Phase 3       Phase 4      Phase 5       Phase 6
核心逻辑    →   结构设计    →     草稿    →    审校    →    打磨    →    定稿
    │               │               │            │            │             │
    ├─ 调研         ├─ 脚手架       │            │            ├─ 口语检验   │
    ├─ 思考判断     ├─ 弧线/权重    │            │            ├─ 密度压缩   │
    └─ 找核+攻核    └─ 读者视角     │            │            ├─ AI清除     │
                     检查           │            │            └─ 反风格     │
                                    │            │                          │
                              writing-dna    review-       humanizer     模式
                              + 模式文件     protocol      (4-pass)     检查清单
```

**Phase 1：核心逻辑** — 调研受众，识别核心张力（"这篇改变读者什么认知？"），用四把铲子挖出核心观点，然后攻击它。核碎了就老实说。产出：Research Summary + 核心陈述。

**Phase 2：结构设计** — 为核找跨学科承重类比（去掉它整篇塌才算数）。选叙事弧线，刻意分配权重（重点 40%，背景 5%）。骨架呈给你确认再动笔。

**Phase 3：草稿** — 加载 15 条写作原则 + 模式模板，逐节写。每个细节挂回脚手架，读者随时知道自己在看什么。

**Phase 4：审校** — 四个维度：结构（对不对模板？）、逻辑（每个 claim 有证据吗？）、事实（数字对吗？）、视角（是给读者写的还是给自己写的？）。必修项全过才能继续。

**Phase 5：打磨** — 四轮依次跑。第 1 轮口语检验优先级最高——后面所有修改都要再过口语关。第 2 轮五维密度压缩。第 3 轮 AI 痕迹清除（黑名单 + 结构检测 + 翻译腔免疫）。第 4 轮八项反风格检查。

**Phase 6：定稿** — 跑模式专属的自检清单。全过才输出。

### 设计决策

| 决策 | 选择 | 原因 |
|------|------|------|
| 找核在结构之前 | Phase 1 和 Phase 2 分开 | 不知道真正要说什么就没法搭结构。大多数 AI 写作工具跳过这步——所以产出结构正确但思想空洞 |
| 4 轮打磨而非 2 层 | 合并两套独立的反 AI 方法论 | 口语检验能抓到黑名单漏掉的。黑名单能抓到口语检验漏掉的。合起来比单独用任何一套覆盖面都大 |
| 按模式调声音温度 | 每种模式自己的 °C 设定 | 小红书 18°C 读起来像教科书。技术文档 35°C 读起来像博客。读者对不同场景的期待不同 |
| 15 条原则而非 9 条 | 新增场景先行、渐进揭示、让步弯道等 | 原来 9 条偏产品写作。新增的 6 条（来自 ljg-writes 方法论）补上了观点文、创意写作和中文写作的深度 |
| 强制调研门禁 | Phase 1 不可跳过 | 没有调研，AI 写作天生就是泛泛的——没有具体数据、没有受众理解、没有独特角度。门禁让这一步不可协商 |

## 快速开始

```bash
git clone https://github.com/d-wwei/great-writer.git
```

**方式 A — Skill 目录**（Claude Code、Codex、Cursor、Windsurf）：
把 clone 的目录指向你的 Agent 技能路径。

**方式 B — 单文件**（ChatGPT、自定义管线）：
把 `great-writer-bundled.md` 注入 system prompt 或上下文。

然后跟你的 Agent 说：

```
"写一篇公众号文章，关于我们的新产品"
"Write a blog post about our launch"
"帮我润色这篇文章"
"Write a README for this repo"
```

Skill 自动识别写作类型，加载对应模式。
